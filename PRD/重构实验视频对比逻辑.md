# 重构实验视频对比逻辑 + OOM 自动等待

## 需求背景

当前 `compare_all.py` 对每个视频运行两次 DiffuEraser 推理（Baseline + Text-Guided），生成 BL vs TG 对比视频并叠加 metrics 分数。这与现在"每个实验对应唯一配置"的实验设计冲突。

## 修改需求

### 1. 视频对比模式 (`compare_all.py`)
- **取消** BL/TG 双推理，改为单次 DiffuEraser 推理（使用实验配置的 gs/prompt/blend/dilation）
- 对比视频改为 **GT vs 实验结果**：左侧 GT，右侧实验输出
- 画面**不叠加**任何分数/prompt 文字，只保留简洁标签（"GT" / "Experiment"）
- Metrics 和 VBench 评估保留单组输出（不再有 BL/TG 两组）

### 2. 调度脚本 (`run_20exp_unified.sh`)
- GPU 动态分配：运行前检查所有 GPU 显存，选择空闲 ≥18GB 的 GPU
- OOM 自动等待：遇到 OOM 不跳过，等待 60s 后重新选 GPU 重试（最多 3 次）
- 自动清理 OOM 产生的空目录
- 限制最大并行数为 6
- 已完成实验（有 summary.json）自动跳过

### 3. 清理与重跑
- 删除现有 20 个实验结果目录
- 使用 nohup 后台运行全部实验
